Learning Path

四段式学习路线

先能安全地问，再能稳定地改，最后把 Claude Code 接进你的真实开发流。

1

会启动

掌握 `claude`、`claude -p`、`--continue`、`--resume`、`--model` 和安装/登录/更新命令。

2

会控场

用 `/plan`、`/permissions`、`/model`、`/effort`、`/compact`、`/rewind` 管住范围、成本和上下文。

3

会交付

练会探索代码库、修 bug、重构、补测试、生成 PR 说明和跑验证命令。

4

会扩展

把 MCP、Hooks、Skills、Subagents 和插件用于真实工具链，而不是把所有上下文都手动粘贴进聊天。

Command Atlas

命令速查与过滤

按使用场景过滤。点击「复制」可把命令放进剪贴板；具体版本以你的 `claude --help` 和官方文档为准。

claude

在当前项目目录启动交互式 Claude Code 会话，适合持续探索和代码修改。

claude "explain this project"

带着第一条任务进入交互模式，适合把意图和项目上下文一起交给 Claude。

claude -p "summarize this log" --output-format json

非交互模式，执行完即退出。适合脚本、CI、日志摘要和结构化输出。

cat logs.txt | claude -p "find the root cause"

把文件或命令输出通过管道喂给 Claude，避免复制粘贴长日志。

claude -c

继续当前目录最近一次对话。非交互脚本里也可以组合 `-c -p`。

claude -r "auth-refactor" "finish this PR"

按会话 ID 或名称恢复上下文，适合跨天继续同一条工作线。

claude install stable && claude auth status --text

安装稳定版并检查登录状态。也可以用 `claude update` 更新已有版本。

claude --add-dir ../apps ../lib

给当前会话增加额外可访问目录，适合 monorepo 或跨仓库排查。

claude --model sonnet --max-turns 5

启动时指定模型和最大 agentic turn 数，适合限制自动化任务边界。

claude --permission-mode plan

从计划模式开始，让 Claude 先读代码和给方案，再决定是否进入修改阶段。

claude mcp add --transport http notion https://mcp.notion.com/mcp

通过 MCP 接外部工具。远程服务优先 HTTP；本地工具可用 stdio。

claude plugin install code-review@claude-plugins-official

安装插件扩展能力。团队场景要先确认插件来源和权限边界。

/help

查看当前环境可用命令。不同平台、计划和环境显示的命令可能不同。

/status 或 /usage

查看版本、模型、账号、连接和使用统计。`/cost`、`/stats` 是 `/usage` 的别名。

/compact 保留当前重构目标和测试结果

压缩上下文但继续同一会话。想完全开始新对话时用 `/clear`。

/rewind

打开 checkpoint 菜单，选择恢复代码、恢复对话或从某一点总结上下文。

/config

打开设置界面，管理主题、模型、输出风格、权限和其他偏好。

/model sonnet 与 /effort high

会话中切换模型或推理强度。复杂设计、疑难 bug 可提高 effort。

/permissions

管理 allow、ask、deny 规则，减少重复提示，同时阻止危险命令或敏感文件读取。

/plan 修复登录失败但先不要改代码

直接进入计划模式。适合需求不大但要先看代码、列风险和测试方案的任务。

/debug npm test 失败，帮我读日志定位根因

开启调试工作流，适合 CLI 自身问题或项目运行错误的系统排查。

/review 或 /security-review

本地审查 PR 或当前改动。安全相关变更用 `/security-review` 单独过一遍。

/simplify focus on reducing duplication

对最近改动做复用、质量和效率检查，再应用小范围简化。

/batch migrate old API clients to the new wrapper

把大型改动拆成多个工作单元并行执行。适合范围清晰的大规模迁移。

/loop 5m check if the deploy finished

定期重复执行提示。长期任务可进一步用 `/schedule` 管理 routines。

/agents

管理自定义 subagent。内置 Explore、Plan、general-purpose 会按任务自动使用。

/skills

查看技能。Claude Code 的自定义 commands 已并入 Skills 机制。

/hooks

查看和管理 Hook 配置。常见用途是阻断危险 shell 命令、自动跑测试或补充上下文。

Permission Modes

权限模式怎么选

权限模式决定 Claude 在编辑文件、运行命令或发起网络请求前是否暂停询问。CLI 内可用 `Shift+Tab` 切换常见模式。

default

默认最稳：读操作自动进行，写文件和命令执行通常会请求确认。适合新项目和敏感仓库。

plan

只读分析和方案设计。适合先摸结构、看风险、写执行计划，然后再进入实现。

acceptEdits

自动接受文件编辑和常见文件系统命令。适合你盯着 diff 快速迭代。

auto

更少打断，配合后台安全检查。适合长任务，但要给清晰目标、验证命令和边界。

dontAsk

只允许预先批准的工具自动运行。适合 CI、脚本化或锁得很严的工作环境。

bypassPermissions

跳过权限层，只适合隔离容器或虚拟机。不要在真实个人目录、生产仓库或含密钥的环境里使用。

Daily Workflows

把命令串成真实工作流

Claude Code 的关键不是「问一个答案」，而是让它读、改、跑、验、解释，形成可复用闭环。

陌生代码库探索

  • 进入项目根目录，启动 `claude`。
  • 先问「这个项目做什么、入口在哪里、测试怎么跑」。
  • 用 `@文件路径` 指定关键文件，减少无关上下文。
  • 要求输出架构图、模块职责和后续问题清单。

Bug 修复

  • 给出复现命令、错误日志和期望行为。
  • 先用 `/plan` 或 plan 模式让它列假设。
  • 确认修复方向后让它改最小范围。
  • 运行同一条复现命令和相关测试。

重构与清理

  • 先锁行为：测试、快照、输入输出样例。
  • 明确「不改公共 API」或「只改某目录」。
  • 每轮只做一个 smell：重复、命名、边界或依赖。
  • 用 `/simplify` 做第二遍质量检查。

PR 与代码审查

  • 让 Claude 读 `git diff` 后先列风险。
  • 用 `/review` 或 `/security-review` 专项审查。
  • 要求 PR 描述包含改动、原因、验证和风险。
  • 提交前跑 lint、typecheck、tests。

MCP 工具接入

  • 只接你信任的数据源和工具。
  • 远程服务优先 HTTP transport。
  • 用项目 `.mcp.json` 分享团队需要的服务器。
  • 用 `@server:resource` 引用 MCP 资源。

Hooks 自动化

  • 用 PreToolUse 阻断危险命令。
  • 用 PostToolUse 在编辑后跑局部测试。
  • 脚本必须校验输入并跳过密钥文件。
  • 长任务用 async hook，避免阻塞会话。
Prompt Templates

可直接复制的训练提示词

把目标、边界、验证方式写清楚，Claude Code 才能稳定地自己检查结果。

代码库总览
请先只读分析这个项目：
1. 说明它做什么、主要入口在哪里
2. 列出构建、测试、部署命令
3. 标出最需要保护的目录和配置
4. 给我一个 30 分钟上手路线
暂时不要修改文件。
Bug 修复
我遇到这个错误：[粘贴错误]
复现命令是：[命令]
请先分析 3 个可能根因，说明你要读哪些文件。
确认最可能根因后，做最小改动并运行复现命令验证。
安全改动
请审查当前 diff 的安全风险，重点看：
认证鉴权、敏感信息、命令注入、路径遍历、依赖供应链。
先列 findings 和文件行号，再给最小修复建议。
