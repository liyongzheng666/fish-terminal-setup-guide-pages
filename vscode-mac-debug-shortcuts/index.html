断点操作 · 媲美 VS2022

普通断点、条件、命中数、函数断点、Logpoint、触发器 —— VS2022 上常用的一整套，VSCode 几乎都内置了，只是少有人知道在哪。

Hit Count Triggered by Logpoint Watchpoint

VS2022 → VSCode 断点功能对照： 条件断点、命中次数、函数断点、跟踪点（VSCode 叫 Logpoint）、异常断点、数据断点（CodeLLDB 的 Break on Value Change）、触发器断点（Triggered by，VSCode 1.86+）全部原生支持。在已有断点上右键 → Edit Breakpoint 或按 ⌘+K+⌘+E，所有选项都在一个弹层里。

✎

Logpoint 语法

花括号包变量，不暂停只打印到 Debug Console：

// 在 BOPAlgo_PaveFiller::Perform 第一行加 Logpoint
Perform called: tools={myArguments.Extent()},
                 fuzzy={myFuzzyValue}

等价于到处写 std::cout，但**不需要重新编译**。

#

Hit Count 表达式

命中次数支持运算符，循环排查神器：

= 10仅第 10 次命中时断
> 5第 5 次起每次都断
% 100每 100 次断一次

▣

一次设置 · 永久记住

  1. 启动一次调试，让左侧 Run & Debug 视图出现
  2. 右键 Variables 或 Watch 标签 → Move View to New Window
  3. 把弹出的小窗口拖到副屏 / 主屏右侧
  4. "window.restoreWindows": "all" 会让下次启动 VSCode 时自动复原这个布局

Config

对应的两个 JSON 文件

放在 ~/Library/Application Support/Code/User/，VSCode 启动后自动加载。

⚙

settings.json · 调试体验

{
  "debug.inlineValues": "on",
  "debug.showInlineBreakpointCandidates": true,
  "debug.showVariableTypes": true,
  "debug.autoExpandLazyVariables": true,
  "debug.toolBarLocation": "docked",
  "debug.openDebug": "openOnDebugBreak",
  "window.restoreWindows": "all",
  "window.nativeTabs": false
}
⌨

keybindings.json · 快捷键映射

// 节选 · 完整版在仓库 README 里
[
  { "key": "cmd+'",
    "command": "workbench.action.debug.stepOver" },
  { "key": "cmd+;",
    "command": "workbench.action.debug.stepInto" },
  { "key": "cmd+shift+;",
    "command": "workbench.action.debug.stepOut" },
  { "key": "cmd+enter",
    "command": "workbench.action.debug.continue" },
  { "key": "cmd+\\",
    "command": "editor.debug.action.toggleBreakpoint" }
]

C++ 配置补充： 这套快捷键搭配 vadimcn.vscode-lldb（CodeLLDB）使用最顺手。 在工程的 .vscode/launch.json 里把 "type" 设成 "lldb"， 再用 initCommands 注入 step-avoid-regexp 跳过 std:: / boost:: / Qt 内部栈帧， 单步效率会比直接进 STL 高一截。
