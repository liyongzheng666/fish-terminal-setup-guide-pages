Fish Terminal Setup 快捷键图鉴

这页不是泛泛的 Fish Cheat Sheet，而是专门按照 Terminal-setup 仓库里的 Fish 路线整理出来的“实装版”速查表。 我把仓库真正接入的组件、仓库直接给出的快捷键、以及我补充核对过的上游默认按键分层标清了。

结论先说在前面：这个方案并没有额外装 fisher、oh-my-fish 之类的插件管理器， 它走的是 Fish 原生能力 + fzf/zoxide + 一批 CLI/TUI 工具 的轻量路线。 真正需要背键位的重点是 Fish、fzf、lazygit、btop 和 delta。

范围判断

用户问的是 “Fish 这个技术选型下所有插件的基础快捷键”。所以我先按仓库实现方式做了三个判断：哪些是 Fish 路线真实接入的组件，哪些组件真的存在键盘快捷键，哪些更适合归到“输入式捷径”。

1. 这不是 Oh My Fish 套餐

仓库的 Fish 配置只在 config.fish 里初始化 starship、fnm、zoxide、fzf，并追加一组 Fish abbr。没有 fisher、oh-my-fish 或额外 Fish 插件仓库。

仓库直出 零插件管理器

2. 真正需要背键位的组件很集中

键位主要集中在 Fish 原生交互、fzf、lazygit、btop 和 delta。像 starship、ghostty、fnm 在这个仓库里并没有额外自定义快捷键。

键盘快捷键 非所有组件都有键位

3. zoxide / abbr 更像“输入式快捷入口”

z proj、zi、ls→eza、lg→lazygit 这种更适合归类为 typed shortcuts。它们不是按键映射，但在日常使用里同样应该背。

命令捷径 Fish abbr

Fish 路线真实接入清单

这一段只回答“它到底装了什么、有没有快捷键”这件事，不把说明文档里顺手提到的工具和真正接入到 Fish 的内容混在一起。

Fish Shell

这套方案的 shell 本体。自动补全、autosuggestion、syntax highlighting 都是 Fish 原生能力，不是额外插件。

有基础键位 上游默认

fzf

仓库显式执行 fzf --fish | source，并设置了 FZF_DEFAULT_COMMAND、FZF_CTRL_T_COMMAND、FZF_ALT_C_COMMAND。

有基础键位 仓库直出

zoxide

仓库显式执行 zoxide init fish | source，并把 cd 写成了 abbr -a cd "z"。更像命令捷径，而不是键位。

命令捷径 仓库直出

Fish Abbreviations

仓库把 ls、ll、lt、cat、find、grep、top、lg、cd 全部写成了 Fish abbr。

命令捷径 仓库直出

lazygit / btop / delta

这些不是 Fish 插件，但都属于 Fish 技术选型下实际安装的交互式工具。它们的基础键位值得一起整理成速查表。

有基础键位 上游默认

Ghostty / Starship / fnm

仓库会安装并配置，但在 Fish 路线里没有额外自定义按键。它们更像显示层、提示符和版本管理层。

无额外键位 仓库直出

键盘快捷键总表

这里收的是“真按键”。搜索框会同时匹配组件名、键位和描述。默认展示仓库 Fish 路线里最值得背的快捷键组。

命令型捷径

这部分不是“按键”，但在 Fish 方案里一样值得背，因为仓库是用 abbr 和轻量函数来提速的。实际用起来，它们和快捷键一样高频。

Fish Abbreviations

这些都是仓库直接写进 config.fish 的缩写。它们会在交互式 shell 中自动展开。

ls 展开为 eza --icons --group-directories-first
ll 展开为 eza -la --icons --group-directories-first
lt 展开为 eza --tree --icons --level=2
cat 展开为 bat
find 展开为 fd
grep 展开为 rg
top 展开为 btop
lg 展开为 lazygit
cd 展开为 z，把目录跳转交给 zoxide

zoxide / fnm / SSH 切换

这组不是按键映射，但它们是 Fish 方案里真正提升效率的“输入式快捷入口”。

z proj 按模糊关键字跳到最常用目录；README 例子是 z proj / z doc
zi 调用 zoxide 的交互式目录选择
fnm install --lts 安装最新 LTS Node；Fish 里用 fnm env --use-on-cd --shell fish 接入自动切换
fnm use 22 当前 shell 切到 Node 22；配合 .node-version 可自动切
set-ssh-key my-key-name 清空 agent 并载入指定私钥；仓库在 Fish 配置里内置了这个函数

装了，但不需要背额外快捷键

这些组件确实属于这套 Fish 技术选型，但仓库没有再给它们叠一层额外键位，所以不用硬记“快捷键表”。

Ghostty

仓库只提供主题、字体、字号和窗口 padding 配置，没有自定义 Ghostty keybindings。

Starship

它负责 prompt 外观，不负责按键交互；这套方案只是把它初始化到 Fish 里。

bat / eza / fd / rg / jq / tldr

它们是命令替代工具，不是 TUI，因此更该记的是命令名或 abbr，而不是按键。
